After setting up the dependencies we need to tackle some key concepts :

What is a web service?


The most common web services we use on a daily basis are :
  1. User Authentication Service : Handles sign-ups, logins, and session management.
  2. Payment Processing Service : Manages transactions and payments securely.
  3. Notification Service : Sends emails, SMS, or in-app notifications to users.
A service is like a unit that does one job really well and is ready when needed. In an app or website, itâ€™s a small part that works on its own but helps the bigger system run smoothly.
A service is often hosted in one or multiple servers.
The same service can be reused across different platforms too like the Web, Android or iOS!

How can we use a service ?


To consume web services, we send requests through an API (Application Programming Interface) and receive responses. The request is like asking a question or providing instructions, and the API helps pass that request to the web service. The service then processes it and sends back a response, which contains the data or result we need. This interaction allows different systems or apps to communicate seamlessly and share information.
